## 1. 项目定位

- 游戏名：原野契约 Demo。
- 类型：单机网页 Demo，2D 宠物收集、培养、合成、回合制战斗。
- 技术栈：React 19 + TypeScript + Vite。
- 运行方式：浏览器打开即可游玩，进度保存在本地浏览器 `localStorage`。
- 当前部署：GitHub Pages，推送 `main` 后由 `.github/workflows/deploy.yml` 自动发布。
- 当前语言：默认中文，游戏内可切换日语。

## 2. 本地运行与部署

常用命令：

```bash
./.tools/bin/npm run dev
./.tools/bin/npm run build
GITHUB_PAGES=true ./.tools/bin/npm run build
```

普通本地环境也可以使用：

```bash
npm run dev
npm run build
```

注意事项：

- Codex 环境里之前出现过系统 `node/npm` 不可用的问题，目前项目内优先使用 `./.tools/bin/npm` 和 `./.tools/bin/node`。
- `vite.config.ts` 根据 `GITHUB_PAGES=true` 设置部署 base 为 `/wildbond-demo/`。
- GitHub Pages 部署成功后，线上 HTML 应指向类似 `/wildbond-demo/assets/index-*.js` 的资源。
- 在 Codex 沙箱中，启动本地 Vite、无头 Chrome、`git add/commit/push` 可能需要提升权限。

## 4. 当前核心玩法规则

### 宠物体系

- 元素属性：火、水、森、土、风。
- 阶段名称：初级、中级、高级、神兽。
- 每个属性 10 只宠物：4 只初级、3 只中级、2 只高级、1 只神兽。
- 宠物个体等级：Lv1 到 Lv10。
- 初始选择 5 只宠物之一，默认获得 Lv3 稀有个体。
- 宠物稀有度：弱小、普通、稀有。
- 稀有度倍率：弱小 0.9，普通 1.0，稀有 1.1。
- 野外敌人稀有度概率：稀有 10%，普通 50%，弱小 40%。

### 基础属性

当前属性项：

- HP
- 攻击
- 防御
- 速度
- 暴击

速度决定行动顺序；速度相同则玩家方优先。暴击为百分比，范围 10% 到 50%，触发后本次攻击伤害 1.5 倍。

元素倾向：

- 火：攻击、暴击偏高，防御偏低。
- 水：HP、暴击偏高，速度偏低。
- 森：防御、速度偏高。
- 土：HP、防御偏高，速度偏低。
- 风：速度、攻击偏高，防御偏低。

### 属性克制

位置：`src/game/balance.ts`

- 火克森。
- 水克火。
- 森克水。
- 土克风。
- 风克土。
- 优势攻击造成 120% 伤害。
- 劣势攻击造成 80% 伤害。
- 同属性或无克制关系造成 100% 伤害。
- 属性克制倍率会和攻击、防御、技能倍率、暴击等基础计算一起生效。

### 技能与 AP

- 每只宠物最多 3 个技能。
- 所有宠物第 1 个技能都是 0AP 基础攻击，保证每回合可行动并积攒 AP。
- 战斗单位初始 AP 为 1，每轮结束最多恢复到 3。
- 技能目标类型：敌方单体、我方单体、自身、敌方全体、我方全体。
- PC 鼠标 hover 和手机长按技能词条都会显示技能详细说明。

当前元素技能定位：

- 火：单体/群体高伤害、灼烧。
- 水：治疗、控速、护盾、群体恢复。
- 森：防御、再生、缠绕、群体续航。
- 土：护盾、石甲、破甲、承伤。
- 风：高速攻击、加速、破甲、群体速度辅助。

### 战斗操作

当前最新版战斗交互已改为“目标优先”：

- 先点击敌方目标，再点击攻击技能，即攻击该目标。
- 群体攻击不受当前目标影响，对敌方全体生效。
- 捕获是独立按钮，捕获当前选中的敌方目标。
- 增益/治疗类单体技能点击后，画面中央出现提示，让玩家选择我方目标。
- PC 按 ESC 可取消增益/治疗目标选择。
- 手机端点击其他技能或战斗其他区域可取消。
- 旧版“点击技能后弹出敌方目标子按钮”的流程已移除。

### 防御

- 所有宠物可选择防御。
- 防御后本回合不攻击。
- 被攻击时伤害降低到 1/3。

### 捕获

位置：`src/game/combat.ts`

- Boss 不可捕获。
- 只要仍有敌方野生宠物，每场战斗可捕获多只。
- 敌方 HP 越低，捕获概率越高。
- 阶段越高、等级越高越难捕获。
- 捕获上限按阶段不同：初级最高 80%，中级最高 55%，高级最高 35%，神兽最高 20%。
- 当前野外不会出现高级和神兽，因此高级/神兽捕获主要是规则兼容。
- 目标陷入灼烧、湿滑、缠绕、破甲等状态会增加捕获概率。
- 多次捕获失败会给该目标少量尝试加成。

### 经验与升级

- 所有存活出战宠物在战斗胜利结算时获得经验。
- 捕获成功的敌人也算作胜利贡献，进入经验结算。
- 多敌人战斗经验按被击败/捕获敌人的经验累加。
- 同阶段同等级参考值：敌方 LvN 提供约 N×10 经验。
- 升级需求：Lv1 到 Lv2 需要 10 经验，Lv9 到 Lv10 需要 90 经验。
- 达到 Lv10 后不再继续积累经验。

### 战斗掉落

- 每场战斗胜利后，30% 概率获得治疗果 x1。
- 每场战斗胜利后，60% 概率获得捕获石 x1。
- 数量无上限。

## 5. 合成、分解、强化

### 合成阶段

位置：`src/game/fusion.ts`

- 合成必须使用两只同阶段、Lv10 的宠物。
- 神兽不可继续合成。
- 初级 + 初级 -> 中级，成功率 80%。
- 中级 + 中级 -> 高级，成功率 50%。
- 高级 + 高级 -> 神兽，成功率 30%。
- 同属性合成时，结果锁定为下一阶段同属性随机宠物。
- 不同属性的初级/中级合成时，结果为下一阶段全池随机。
- 高级合成神兽要求同属性且两只高级宠物必须不同种。

合成失败：

- 两只材料消失。
- 留下两只材料中随机一只的 Lv1 个体。
- 合成成功或失败都会在界面显示短提示。

合成结果稀有度：

- 稀有 20%。
- 普通 40%。
- 弱小 40%。

### 分解

- 出战中的宠物不可分解，按钮应为灰色不可点击。
- 入库宠物可分解。
- 初级分解获得 1 分解水晶。
- 中级分解获得 2 分解水晶。
- 高级分解获得 3 分解水晶。
- 神兽暂时不可分解。
- 高级强化过的宠物分解时，会返还基础 3 点 + 已投入强化水晶。

### 强化

- 只有高级宠物可强化。
- 神兽暂时不可强化。
- 最高强化 +3。
- +1 成本 3 水晶，成功率 80%。
- +2 成本 6 水晶，成功率 50%。
- +3 成本 10 水晶，成功率 30%。
- 强化失败保留当前强化等级，但消耗水晶不返还。
- 每次强化成功增加基础属性倍率，当前为每级 +8% 属性，暴击每级 +3。

## 6. 地图与解锁

地图配置在 `src/data/maps.ts`。

当前 5 张地图：

- 青芽草原：森系，80×60，营地 8,8，Boss 72,52。
- 潮汐海岸：水系，90×64，营地 8,9，Boss 82,56。
- 赤砂火山：火系，84×70，营地 9,8，Boss 76,62。
- 岩牙峡谷：土系，96×72，营地 9,9，Boss 88,64。
- 风鸣高地：风系，100×76，营地 9,9，Boss 92,68。

解锁规则在 `src/game/state.ts`：

- 初始解锁青芽草原。
- 击败青芽草原 Boss 后解锁潮汐海岸。
- 首次初级合成成功后解锁赤砂火山。
- 击败潮汐海岸和赤砂火山 Boss 后解锁岩牙峡谷。
- 首次中级合成成功且击败岩牙峡谷 Boss 后解锁风鸣高地。

遇敌机制：

- 地图中营地附近只会出现低等级初级野怪。
- 离营地越远，敌人等级越高，远区才会出现中级野怪。
- 野外不出现高级和神兽。
- 10 到 15 次有效移动内会显著提高遇敌概率，第 15 次保证遇敌。

## 7. Boss 与强化挑战

位置：`src/game/combat.ts`

- 普通 Boss 队伍：当前地图属性神兽 Lv10 一只 + 同属性高级 Lv10 两只。
- Boss 均为普通标准稀有度，不可捕获。
- 普通 Boss 当前基础倍率为 `0.9`。
- 强化挑战上限 3 次。
- 强化挑战倍率当前为：
  - 普通 Boss：0.9
  - 强化 +1：1.2
  - 强化 +2：1.3
  - 强化 +3：1.4
- 设计意图：没有神兽时，三只高级 Lv10 应能打过普通 Boss，但需要操作；有神兽后普通 Boss 明显更轻松，强化挑战更适合后期队伍。

## 9. 美术资源规范

当前最新规范见 `美术资源规范.md`。

宠物模型：

- 当前正式宠物图使用 `512×512 RGBA PNG`，真实透明背景。
- 路径：`src/assets/pets/latest/{element}/{pet-id}.png`。
- 绑定表：`src/data/petArtwork.ts`。
- 游戏内使用 `object-fit: contain` 等比例缩放。
- 替换宠物图只绑定宠物内部 ID，不应影响战斗、捕获、升级、合成、存档。

初始选宠图：

- 路径：`src/assets/starters/`。
- 只影响开局选择界面，不等同于全局宠物模型。

仍需注意：

- 旧的程序化 64×64 宠物像素文件还在 `src/data/*PetPixelArt.ts`，不要在未审计前直接删除。
- 地图瓦片、人物、UI 图标等仍以 64×64 像素体系为基础。
- 如果后续替换地图、人物、建筑、环境贴图，需要重新确认锚点、遮挡、手机显示、战斗显示和资源加载体积。

## 13. 已知注意事项

- 线上发布后 GitHub Pages 可能有短暂缓存；可在 URL 后加 `?v=提交号` 强制刷新。
- 当前构建会出现 Vite chunk 超过 500KB 的警告，这是资源和逻辑集中打包导致的警告，不是构建失败。
- 50 张宠物 PNG 已纳入正式构建，后续资源继续增加时需要关注首屏加载体积。
- 宠物 ID 是存档和图片绑定的核心键。改名可以改中文名，但不要随意改 `id`。
- 神兽当前不可强化、不可分解、不可野外捕获。
- 普通野外只出现初级和中级。高级和神兽主要通过合成、Boss 队伍出现。
- 开发中如果改动 `pets.ts`、`skills.ts`、`balance.ts` 或 `combat.ts`，必须同步跑玩法回归和至少一次浏览器战斗 QA。
- 如果改 UI 布局，必须跑 PC 和手机响应式 QA。
- 如果替换宠物图片，必须跑宠物图像运行时 QA，并人工确认透明边缘、锚点、大小和战斗显示。

## 14. 推荐继续开发流程

1. 读取本文件和相关设计文档。
2. 确认当前工作区是否干净：`git status --short`。
3. 启动本地服务并验证目标问题。
4. 小步修改，避免一次改动跨越玩法、UI、资源三大系统。
5. 根据改动类型运行对应 QA。
6. `GITHUB_PAGES=true ./.tools/bin/npm run build` 通过后再提交。
7. 推送 `main` 后检查 GitHub Actions 部署是否成功。
8. 线上打开 `https://loki-zhao.github.io/wildbond-demo/` 做最终冒烟测试。
