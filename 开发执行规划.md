# Demo开发执行规划 v0.1

状态：执行中  
目标：完成一个可在本地浏览器打开试玩的 React + TypeScript + Vite 单机网页 Demo

## 1. 本次开发范围

本次 Demo 聚焦完整核心循环：

- 2D 地图移动。
- 暗雷遇敌。
- 回合制战斗。
- 主角 + 最多 3 只宠物协战。
- 捕获宠物。
- 队伍与仓库管理。
- 宠物图鉴。
- 同阶段 Lv10 宠物合成。
- 5 张地图解锁与首领挑战。
- 浏览器本地保存进度。

## 2. 暂不开发的内容

以下内容不进入第一版 Demo：

- 联网账号系统。
- 玩家交易、公会、聊天。
- 真实服务器存档。
- 复杂装备系统。
- 宠物进化树动画。
- 正式商业级美术资源。

## 3. 技术路线

| 模块 | 方案 |
|---|---|
| 前端框架 | React |
| 语言 | TypeScript |
| 构建工具 | Vite |
| 图标 | lucide-react |
| 状态管理 | React useState/useMemo |
| 保存 | localStorage |
| 地图 | CSS Grid Tile 地图 |
| 战斗 | TypeScript 纯函数 + React 状态 |

## 5. 数据落地方式

- 宠物设定来自《宠物与技能设定.md》。
- 地图设定来自《地图与开发计划.md》。
- 技能与数值会转成 TypeScript 常量表。
- 宠物个体会在捕获或合成时生成唯一 ID、个体 Lv、当前 HP、经验值。

## 6. UI 结构

首屏直接进入游戏，不做营销落地页：

- 左侧：地图探索区域。
- 中间/右侧：当前队伍、地图状态、背包信息。
- 底部：事件日志。
- 顶部：地图切换、保存、重开入口。
- 弹层/面板：战斗、仓库、合成、图鉴、地图进度。

## 7. 调试验收

完成后我会自行检查：

- 项目可以构建。
- 本地开发服务器可以启动。
- 页面无明显运行错误。
- 玩家能移动并触发暗雷。
- 战斗能胜利、失败、捕获。
- 捕获后宠物进入队伍或仓库。
- 队伍最多 3 只宠物。
- 初始体 Lv10 + 初始体 Lv10 可合成进化体 Lv1。
- 进化体 Lv10 + 进化体 Lv10 可合成完全体 Lv1。
- 完全体不可继续合成。
- 五张地图可按条件解锁。
- 浏览器刷新后进度保留。

## 8. 遇到设定冲突时的处理

如果开发过程中出现以下情况，会暂停并向你确认：

- 需要改变已确认的五种属性。
- 需要改变宠物进化阶段与个体 Lv 的关系。
- 需要改变合成规则。
- 需要删除或重命名大量宠物、技能、地图。
- 需要改变 Demo 从单机网页变成其他形态。

普通数值微调、布局调整、代码组织、Bug 修复由我直接处理。
